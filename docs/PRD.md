# senior-travel-ai-agent
# 시니어 여행 동선 설계 AI 에이전트 — PRD (Product Requirements Document)

## 1. 개요

| 항목 | 내용 |
|---|---|
| 프로젝트명 | 시니어 맞춤 여행 동선 설계 에이전트 (가칭 "AI 여행 매니저") |
| 한 줄 정의 | 패키지 여행의 편리함과 자유여행의 자유로움을 결합해, 시니어의 체력·취향을 반영한 여행을 대신 설계해주는 에이전트 |
| 배경 | 이번 여름 부모님이 안 가본 나라(캐나다)로 여행을 계획하면서 겪은 실제 어려움에서 출발 |

## 2. 문제 정의 및 포지셔닝

### 기존 두 선택지의 한계
- **패키지 여행**: 편하다. 하지만 정해진 일정에 맞춰야 해서 자유가 없다.
- **자유여행**: 자유롭다. 하지만 체력과 취향을 모두 고려해 직접 계획하는 게 너무 어렵다.

### 이 프로젝트의 포지셔닝
"패키지 여행의 편리함"과 "자유여행의 자유로움"을 결합해, 시니어가 원하는 여행을 편안하게 즐길 수 있도록 돕는 것이 이 에이전트의 핵심 가치다.

> 편안하다 + 자유롭다 + 취향을 반영한다 + 많이 걷지 않는다 + 일정을 직접 계획하지 않아도 된다

### Target User 재정의
- Before: 많이 걷기 힘든 시니어 (체력 제약 중심의 좁은 정의)
- After: **패키지 여행은 싫지만, 자유여행을 계획하기 어려운 시니어** — 훨씬 넓고 명확한 타겟

### 기존 AI 여행 추천과의 차이
기존 AI는 "여행지 → 관광지 나열 → 총 도보거리" 식으로 **추천**하는 데 그친다. 이 에이전트는 체력·취향 조건을 받아 실제 동선(시간대별 일정, 이동수단, 이동 부담 평가, 숙소 추천 근거)까지 **대신 설계**해주는 것이 핵심 차이다.

## 3. 목표

- **Goal 1**: 인터뷰 형식으로 체력·취향·선호를 구조화해 입력받는다.
- **Goal 2**: 입력된 제약을 만족하는 Day별 동선을 시간대까지 포함해 설계한다.
- **Goal 3**: 각 Day의 이동 부담을 Mobility Score로 정량 평가하고 해석을 제공한다.
- **Goal 4**: 동선 중 필요한 참고 정보(혼잡시간, 편의시설, 현지 문화 등)를 함께 안내한다 (AI Guide).
- **Goal 5**: 숙소를 추천할 때 "왜 이 숙소여야 하는지" 근거를 함께 제시한다 (AI Hotel Manager).

### 성공 기준
- 입력된 제약(도보 시간, 계단 가능 여부 등)을 실제로 만족하는 동선만 결과로 제시되는지
- Mobility Score가 실제 거리·소요시간·휴식 데이터에 근거해 산출되고 해석까지 제공되는지
- 숙소 추천이 단순 목록이 아니라 근거(관광지까지 거리, 지형, 캐리어 이동 부담 등) 비교로 제시되는지

## 4. 대상 사용자 및 사용 시나리오

- **주 사용자**: 패키지는 부담스럽지만 직접 계획은 어려운 시니어 본인 (또는 이를 대신/보완하는 자녀)
- **시나리오**: STEP1 인터뷰 응답 → STEP2 여행 성향 분석 결과 확인 → Day별 동선 자동 설계 → Mobility Score·AI Guide·숙소 추천 근거 확인 → 필요 시 대화로 일정 조정

## 5. 입력 설계 — STEP1. AI 인터뷰

```
1. 연령대: 50대 / 60대 / 70대 이상
2. 하루 몇 시간 정도 걸으실 수 있나요?: 1~2시간 / 3~4시간 / 상관없음
3. 여행 스타일: 자연 / 역사 / 쇼핑 / 맛집 / 공연 / 야경 (다중 선택)
4. 패키지 여행처럼 편한 여행을 원하시나요?: 매우 편한 여행 / 적당한 여행 / 많이 관광하는 여행
5. 자유시간은 얼마나 필요하신가요?: 많음 / 보통 / 상관없음
6. 숙소 이동은 가능한가요?: NO / 1회 가능 / 상관없음
```

## 6. STEP2. AI 여행 성향 분석 (인터뷰 직후 요약 제공)

인터뷰 응답을 곧바로 아래와 같은 형태로 요약해 보여준다 — 사용자가 "AI가 내 성향을 제대로 이해했는지" 먼저 확인할 수 있게 하기 위함.

```
OO님의 여행 성향

여행 스타일: 자연 80% / 맛집 20%
체력 수준: 하루 5,000보 이하 추천
선호 여행: 널널한 일정 / 오전 관광 선호 / 자유시간 중요

AI 추천: "패키지 여행보다 자유시간이 많은 맞춤형 여행을 추천드립니다."
```

## 7. 기능 요구사항

### 7.1 관광지·숙소·식당·이동수단 조사
- 지역 기준으로 관광지, 숙소, 관광지 주변 식당, 지점 간 이동수단 후보를 조사

### 7.2 제약조건 기반 Day별 동선 설계 (핵심 기능)
- STEP1 응답을 제약조건으로 변환해, 이를 만족하는 관광지 조합을 시간대까지 포함해 구성

```
DAY1
09:30 호텔 출발 → 10:00 관광지1 → 12:00 점심 → 13:30 카페(휴식)
→ 15:00 관광지2 → 17:00 호텔 복귀
총 도보거리: 2.8km / 이동 부담: 매우 낮음
```

### 7.3 Mobility Score
- 총 이동거리, 도보거리, 계단, 이동시간, 휴식횟수를 종합해 별점으로 환산하고 산출 근거를 함께 표시
- 연령대 기준 해석 문장 제공 (예: "이 일정은 30대 기준으로는 무리 없지만, 60대 기준으로는 이동 부담이 큰 일정입니다")
- 부담이 큰 Day는 "추천하지 않습니다"와 대안 제시

### 7.4 AI Guide
- 동선 중 참고할 만한 맥락 정보를 함께 제공
```
- 오전에 방문하면 사람이 적습니다.
- 해당 관광지는 엘리베이터가 있습니다.
- 식당까지 도보 5분입니다.
- 캐나다는 팁 문화가 있습니다.
```

### 7.5 AI Hotel Manager
- 숙소를 단순 나열하지 않고, 추천/비추천 근거를 비교로 제시
```
추천하지 않는 숙소: 호텔 A — 관광지와 45분 거리, 언덕 많음
추천 숙소: 호텔 B — 관광지 평균 10분, 기차역 5분, 캐리어 이동 부담 적음
```
- 추천 숙소에는 Agoda, Hotels.com 등 예약 사이트의 **검색 딥링크**를 함께 제공한다. (정식 Affiliate API 연동은 제휴 심사가 필요해 범위 밖으로 두고, 호텔명·도시·체크인/체크아웃 날짜를 쿼리 파라미터로 채운 검색 URL로 "예약 확인하기" 버튼을 연결하는 방식으로 구현한다.)

### 7.6 지도 시각화
- 숙소를 출발점으로 해당 Day 방문지를 경유지 순서대로 지도에 표시, 구간별 거리·소요시간 함께 표시

### 7.7 UI 구성 — Day별 탭 구조
- 트리플 앱 참고: 상단에 "AI 대화" 탭 + "Day 1" ~ "Day N" 탭을 나란히 배치
- AI 대화 탭: STEP1 인터뷰 및 일정 조정용 대화 인터페이스
- Day별 탭: 동선 지도, Mobility Score, AI Guide, 관광지/식당/숙소 정보를 한 화면에 표시

## 8. MVP 우선순위 (구현 범위 고려)

스코프가 넓은 프로젝트이므로, 아래 우선순위로 단계적 구현을 권장한다.

| 우선순위 | 기능 |
|---|---|
| **Core (필수)** | STEP1 인터뷰 → STEP2 성향 분석, 제약조건 기반 Day별 동선 설계(7.2), Mobility Score(7.3) |
| **1차 확장** | AI Hotel Manager(7.5), 지도 시각화(7.6), Day별 탭 UI(7.7) |
| **2차 확장 (여유 있을 때)** | AI Guide(7.4), 관광지 주변 식당 추천 |

## 9. 비기능 요구사항 / 설계 원칙

- Mobility Score는 임의의 인상 점수가 아니라 실제 데이터(거리·계단·휴식)에 근거해 산출되고, 근거가 항상 함께 표시되어야 한다.
- 숙소·동선 추천은 반드시 "왜"를 함께 제시해, 사용자가 추천을 그대로 따르기보다 이해하고 선택할 수 있게 한다.
- 최종 결정권은 항상 사용자에게 있어야 하며, 에이전트는 "확정된 지시"가 아니라 "근거 있는 제안"으로 결과를 제시한다 — 패키지처럼 일방적으로 따라야 하는 구조가 되지 않도록 한다.

## 10. 리스크 및 제한사항

- Mobility Score 산출 기준(가중치)이 불투명하면, 기존 여행 앱과 마찬가지로 "그럴듯해 보이는 점수"에 그칠 위험이 있음
- 기능 범위가 넓어(인터뷰, 동선 설계, 스코어링, 가이드, 숙소 추천, 지도) 전체를 다 구현하기엔 시간이 많이 필요 — MVP 우선순위(8번)에 따라 단계적으로 접근 필요
- 실제 도보 소요시간은 지형(경사 등) 요인을 완전히 반영하기 어려워, 지도 서비스의 평면적 거리·시간 정보에 의존하는 한계가 있음

## 11. 향후 확장 가능성

- Mobility Score와 제약조건 최적화 프레임은 시니어 외에도 이동 제약이 있는 다른 여행자군에 적용 가능
- 부모님이 1차로 일정을 만들고 자녀가 검토·보완하는 협업형 플로우로 확장 가능
